# 프로젝트 이해 참조 워크플로우

모든 작업을 시작하기 전에 아래 문서를 먼저 확인하세요.

1. `docs/codex/PROJECT_KNOWLEDGE.md` 파일을 읽어 프로젝트 전체 구조를 파악합니다.
   - 백엔드 API 엔드포인트 목록
   - DB 스키마 및 엔티티 관계
   - 프론트엔드 컴포넌트 구조
   - 인증/보안 아키텍처
   - 알려진 이슈
// turbo
2. `docs/codex/PROJECT_INDEX.md`, `docs/codex/WORKING_MEMORY.md`를 함께 확인해 현재 작업 범위와 영향 파일을 파악합니다.
// turbo
3. 변경 이력이 있다면 `docs/codex/YYYY-MM-DD/CHANGELOG_YYYY-MM-DD.md`, `WORKLOG_YYYY-MM-DD.md`를 확인합니다.
// turbo
4. 작업 완료 후, 변경 사항이 프로젝트 이해 문서에 영향을 미치면 `docs/codex` 하위 문서를 기준으로 업데이트합니다.
