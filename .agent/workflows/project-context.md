# 프로젝트 이해 참조 워크플로우

모든 작업을 시작하기 전에 아래 문서를 먼저 확인하세요.

1. `docs/PROJECT_KNOWLEDGE.md` 파일을 읽어 프로젝트 전체 구조를 파악합니다.
   - 백엔드 API 엔드포인트 목록
   - DB 스키마 및 엔티티 관계
   - 프론트엔드 컴포넌트 구조
   - 인증/보안 아키텍처
   - 알려진 이슈
// turbo
2. 변경 이력이 있다면 `docs/CHANGELOG_*.md` 파일을 확인합니다.
// turbo
3. 작업 완료 후, 변경 사항이 `PROJECT_KNOWLEDGE.md`에 영향을 미치면 해당 문서를 업데이트합니다.
